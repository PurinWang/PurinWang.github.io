Android Test Tool

除了Coding之外Android Studio也提供了一些基本測試或Debug的工具，有像是模擬猴子不懂隨機控制，也可以寫一些指令循序測試，就讓我們來了解如何使用它吧！
透過Android Studio的ADB(Android Debug Bridge)指令就可使用這些工具可以在終端（手機或模擬器）模擬用戶事件流，像是觸摸事件、滑動事件等，可以不斷模擬重複一樣的行為進行壓力測試。
常見的大概為下列幾項：

  1. Monkey & Monkey Script(MS)
  2. Monkey Runner
  3. Espresso

ADB可以藉由安裝Android SDK Platform Tool取得，在windows上別忘了設定環境變數方便使用。
Monkey Runner除了需要安裝上面的Platform Tool還需要安裝Java JDK及python建立環境。

Monkey & Monkey Script


Monkey之所以稱為猴子是因為整個過程猶如猴子般不懂亂點，但可以制定生成規則，程式會依照規則生成之後並執行。
Monkey Script(MS)為可程式化的腳本類似按鍵精靈，呼叫規定的function而附檔名不限，在Google官方沒公布用法，但從MonkeySourceScript中的註解可以看出使用方法及功能。

Monkey

  • 指定Package
  • 各事件的比例
  • 種子數
  • 間隔時間
  • 設定Log等級或是印出Log於cmd
    Windows：設定好ADB環境變數，確認可以使用後即可開始下monkey指令。

Basic Command

adb shell monkey -p <packagename> --throttle <msec> [-v[-v]...] <count> > <router>
參數 功能
<packagename> 指定android App
<msec> 間隔時間（毫秒）
[-v[-v]…] Log Level
<count> 執行次數
<router> Log存檔位置

Command Parameter

參數 功能
–pkg-blacklist-file <blacklist-file-router> 軟體黑名單，文字檔讓Monkey避開在指定APK軟體執行
–pkg-whitelist-file <whitelist-file-router> 軟體白名單，文字檔讓Monkey在指定APK執行
-ignore-crashes 忽略crash
-ignore-timeouts 忽略timeout
-ignore-security-exceptions 忽略安全例外
-monitor-native-crashes 監控由C/C++引起的crash
-ignore-native-crash 忽略由C/C++引起的crash
-kill-process-after-error 錯誤發生後關閉執行緒
-hprof 測試完成後建立性能報告，通常存放於/data/misc目錄下，分析需要使用 Traceview

Random Command Parameter

參數 功能(%) 備註
–ptc-touch 觸控事件 點選down-up
–ptc-motion 動作事件 點選中同時做其他動作再放開down-somthing-up
–ptc-trackball 軌跡球事件 一系列的滑動或點選
–ptc-nav 基本導航事件 上下左右事件
–ptc-majornav 主要導航事件 Back鍵、Menu鍵等
–ptc-syskeys 系統按鍵事件 系統使用的保留按鍵像Home鍵、音量鍵等
–ptc-appswitch 應用啟動事件 打開Activity相當於打開APP
–ptc-anyevent 其他事件 不常用的按鈕、其他事件等

事件總和必須為100%

Monkey Script(MS)

  • 使用預先寫好的基礎腳本
  • 壓力&重複測試
  • 輸出Log

Basic Command

  1. 首先先寫好一份腳本，腳本內容包含一個Head及Body如下：
type= raw events
count= <event count>
speed= 1.0
start data >>
        <command>
參數 功能
type 指定測試類型通常為一連串的事件(raw events)沒有實質使用
count 事件總數沒有實質使用
speed 設定兩次事件發送的頻率
start data 開始編寫腳本的位置
  1. 將寫好的檔案推至Android主機上並記得路徑
adb push <script name> <router>
參數 功能
script name 腳本名稱並包含路徑
router Android主機上的路徑
  1. 下指令執行腳本和Monkey雷同，但這邊的count是腳本執行次數
adb shell monkey –f <router> --throttle <msec> [-v[-v]...] <count> > <router1>

Monkey Runner


有別於Monkey隨機生成事件或未公開的Monkey Script用法的做法，Monkey Runner可以有更多功能及用法：

  • 多設備控制
  • 特定功能測試
  • 重複回歸測試
  • 可使用module

Environment Setup

必須先配置環境，安裝Pyhon、Java Developer Kit(JDK)、Android Developer Tool(ADT)

Python

  • windows
    到Python 官方網站下載並安裝，再進入系統設置配置環境變數

    加入path後再Cmd下指令
python --version

有跑出版本就代表成功

  • Mac
    可以到官網下載也可以打開Terminal輸入
brew install python

*若無brew可以到Homebrew官網根據指令安裝

Java Developer Kit(JDK)


到Java官網下載並安裝，再進入系統設置環境變數

加入path後再Cmd下指令

java --version

有跑出版本就代表成功
*若不可用可嘗試抽換版本

Android Developer Tool(ADT)


到Android Developer官網下載Android Studio並安裝，進入後下載Tools SDK，再進入系統設置環境變數，而Monkey Runner就在Tools bin裡面

最後再terminal或Cmd內下指令，順利的話可進入編譯器

monkeyrunner

Basic Script

monkeyrunner -plugin<plugin_jar> <program_filename> <program_options>
參數 功能
<plugin_jar> java插件
<program_filename> python腳本名稱
<program_options> 設定program_file旗標或是參數

下monkeyrunner進入編譯器後，根據Android官方說明以下為範例程式碼，主要功能為等待裝置連接後安裝APP，跑指定的Activity並點按指定按鍵最後儲存截圖。

from com.android.monkeyrunner import MonkeyRunner, MonkeyDevice
device = MonkeyRunner.waitForConnection()
device.installPackage('myproject/bin/MyApplication.apk')
package = 'com.example.android.myapplication'
activity = 'com.example.android.myapplication.MainActivity'
runComponent = package + '/' + activity
device.startActivity(component=runComponent)
device.press('KEYCODE_MENU', MonkeyDevice.DOWN_AND_UP)
result = device.takeSnapshot()
result.writeToFile('myproject/shot1.png','png')

monkeyrunner將python當作編譯器因此寫法與python一樣，可以控制迴圈、判斷式重複測試將BUG抓出，如以下程式碼，等待裝置連接之後重複執行點按的指令並在其中delay。

from com.android.monkeyrunner import MonkeyRunner, MonkeyDevice
device = MonkeyRunner.waitForConnection()
for i in range(1,10):
    device.touch(1005,545,MonkeyDevice.DOWN_AND_UP)
    MonkeyRunner.sleep(0.3)
